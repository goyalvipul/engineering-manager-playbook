# 30-60-90 Day Engineering Onboarding Plan

**Purpose:** Provide a clear, metric-driven roadmap for new engineers to achieve full autonomy and impact within their first quarter.

## 🗓️ Phase 1: Days 1-30 (Context & Foundation)
**Goal:** Understand the domain, set up the local environment, and ship to production safely.
* **Week 1:** * Gain access to all systems (AWS, GitHub, Datadog, Jira).
  * Successfully run the microservices stack locally using Docker Compose[cite: 13].
  * Ship a "First Good Issue" (e.g., UI text update, small bug fix) to production via our CI/CD pipeline[cite: 12].
* **Week 2-4:**
  * Complete deep-dive sessions on our Event-Driven Architecture and Feature Flagging systems[cite: 23, 27].
  * Pair program on 3+ tickets with a Senior Engineer.
  * Review 5 PRs authored by teammates to understand our coding standards.

## 🗓️ Phase 2: Days 31-60 (Execution & Ownership)
**Goal:** Operate as a fully contributing member of the agile pod.
* Deliver a medium-complexity epic independently (e.g., creating a new API endpoint, updating a DynamoDB schema)[cite: 12, 13].
* Participate in on-call shadowing; learn how to read CloudWatch logs and Datadog APM traces.
* Lead a technical refinement session during Sprint Planning.

## 🗓️ Phase 3: Days 61-90 (Optimization & Leadership)
**Goal:** Identify technical debt, suggest improvements, and operate with high autonomy.
* Successfully complete a primary on-call rotation.
* Ship a high-impact feature aligned with our quarterly OKRs (e.g., an A/B test implementation)[cite: 34].
* Write and present an ADR for a proposed system improvement or refactor.
* **Reverse Onboarding:** Submit a PR to update this very onboarding document based on what was missing during your experience.
